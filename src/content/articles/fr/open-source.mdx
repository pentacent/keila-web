La plupart des outils de newsletter par email sont entièrement détenus et contrôlés par une seule entreprise.
Keila est différent : Keila est un outil de newsletter Open Source construit avec l'aide
d'une communauté de bénévoles et librement disponible pour tout le monde.

## Que signifie le fait que Keila soit un outil de newsletter Open Source ?

Les logiciels Open Source (également appelés logiciels libres) consistent à accorder
certaines libertés aux utilisateurs que les logiciels _propriétaires_ ordinaires n'offrent pas.

- Vous souhaitez comprendre le fonctionnement interne d'un programme et lire son code source ? Aucun problème !
- Vous souhaitez modifier le code du programme pour mieux l'adapter à votre cas d'utilisation ? Allez-y !
- Vous souhaitez partager le programme avec des amis, des collègues ou le grand public ? Avec les logiciels Open Source, vous le pouvez.

## L'Open Source est-il plus sécurisé ?

Certaines personnes s'inquiètent que le code source des programmes Open Source étant disponible, il soit plus facile de trouver et de créer des exploits dans le logiciel. C'est en fait le contraire ! Trouver des exploits dans les logiciels Open Source n'est pas vraiment plus facile que dans les logiciels propriétaires fermés - le code source n'est même pas nécessaire pour les trouver. Mais d'un autre côté, beaucoup plus de personnes peuvent examiner le code d'un programme Open Source, signaler des vulnérabilités potentielles et les corriger.

Un autre avantage en matière de sécurité des logiciels Open Source est que vous pouvez auditer un
programme et ensuite l'exécuter vous-même - vous ne devez pas faire confiance à un tiers.

## Sous quelle licence Open Source Keila est-il distribué ?

Keila utilise la [GNU Affero General Public License (GNU AGPL), Version 3](https://www.gnu.org/licenses/agpl-3.0.html.en).
Voici quelques-uns de vos droits et obligations sous l'AGPLv3 :

### Droits

- Vous pouvez accéder au code source de Keila et exécuter votre propre instance pour toute
  utilisation que vous jugez appropriée
- Vous pouvez apporter des modifications à Keila et partager vos modifications avec
  d'autres personnes

### Obligations

- Si vous exécutez votre propre instance de Keila, vous devez rendre le code source de Keila
  (y compris toutes les modifications) disponible pour tous les utilisateurs qui interagissent avec lui.
  Cela inclut non seulement les utilisateurs ayant un compte sur votre instance de Keila, mais aussi
  les contacts qui interagissent avec d'autres parties de votre instance de Keila, telles que le
  formulaire d'inscription, la page de désabonnement ou la fonction de suivi des emails.

Pour plus d'informations sur l'AGPLv3, consultez la
[FAQ](https://www.gnu.org/licenses/gpl-faq.html) de la Free Software
Foundation et la [licence complète](https://www.gnu.org/licenses/agpl-3.0.html.en).

## Open Core ? Non merci !

Certains programmes ne sont que partiellement publiés en tant que logiciels Open Source, certains
composants restant propriétaires ou des modules complémentaires payants. Cela est communément appelé
[Open Core](https://en.wikipedia.org/wiki/Open-core_model).

Bien que l'Open Core puisse
être un modèle économique valable pour certains, nous nous engageons à maintenir Keila 100% Open
Source sans composants propriétaires. Nous croyons en le logiciel libre et
ses nombreux avantages pour nos utilisateurs et notre communauté.

La plupart des autres outils de marketing par email tels que Mailchimp, Brevo (Sendinblue) ou
